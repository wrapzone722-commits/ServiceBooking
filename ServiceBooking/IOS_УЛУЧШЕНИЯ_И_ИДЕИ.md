# Анализ и идеи по улучшению iOS-приложения Service Booking

## Что уже есть

- **Услуги** — список, категории, поиск, детали услуги, запись (дата, пост, слот, напоминание, комментарий).
- **Мои записи** — предстоящие / прошедшие / отменённые, отмена, акт (PDF/HTML), Live Activity для «В процессе».
- **Новости** — лента с сервера.
- **Профиль** — редактирование (имя, телефон, авто), система лояльности (баллы, обмен на товары/услуги), тема, компания, помощь, чат с сервисом, контакты.
- **Онбординг** — QR/ручной ввод URL, сплэш, проверка сети.
- **API** — метод `submitRating` есть, но **в интерфейсе рейтинга нет**.

---

## Рекомендуемые улучшения (по приоритету)

### 1. Рейтинг после завершённой записи (высокий приоритет)

**Сейчас:** Бэкенд принимает `POST /bookings/:id/rating`, в приложении кнопки «Оценить» нет.

**Сделать:**
- В модели `Booking` добавить опциональные поля `rating: Int?`, `ratingComment: String?` (и при необходимости поддержать их в API списка записей).
- На карточке **завершённой** записи в «Прошедшие»:
  - если рейтинг ещё не выставлен — кнопка **«Оценить»**;
  - если выставлен — текст вида «Вы оценили: ★★★★☆» (и при желании комментарий).
- По нажатию «Оценить» — модальное окно (sheet): 1–5 звёзд + необязательное поле комментария, кнопка «Отправить» → вызов `APIService.submitRating`, обновление списка записей.

**Бэкенд:** В `toBookingJSON` в `web-console/routes/bookings.js` добавить `rating` и `rating_comment`, чтобы приложение знало, что запись уже оценена.

---

### 2. Локальные напоминания о записи (высокий приоритет)

**Сейчас:** В создании записи есть выбор «Напомнить о записи» (за день, за 2 часа и т.д.), но **уведомление не планируется**.

**Сделать:**
- После успешного создания записи по выбранному `ReminderTiming` запланировать локальное уведомление (`UNUserNotificationCenter`) на момент «дата/время записи минус интервал» (например, за 1 день или за 2 часа).
- Текст: «Напоминание: завтра в 10:00 — Мойка кузова» (или «Через 2 часа — …»).
- При отмене записи — удалять соответствующие pending-уведомления по идентификатору записи.
- Запросить разрешение на уведомления при первом выборе напоминания или при первом создании записи.

---

### 3. «Записаться снова» (средний приоритет)

**Сейчас:** В «Прошедшие» только просмотр и акт.

**Сделать:**
- Для завершённой записи добавить кнопку **«Записаться снова»**.
- По нажатию — переход на вкладку «Услуги» и открытие экрана записи на **ту же услугу** (тот же `Service` по `serviceId`), чтобы пользователь сразу выбрал новую дату/время.

---

### 4. Добавить запись в календарь (средний приоритет)

**Сейчас:** Запись есть только внутри приложения.

**Сделать:**
- На карточке предстоящей записи кнопка **«Добавить в календарь»**.
- Через EventKit: создать событие с названием услуги, датой/временем, длительностью и (по желанию) местом/заметкой — «Запись в [название сервиса]».
- Запросить доступ к календарю при первом нажатии.

---

### 5. Рейтинг в API списка записей (для п.1)

**Сейчас:** `toBookingJSON` в `bookings.js` не отдаёт `rating` и `rating_comment`.

**Сделать в веб-консоли:**
- В `toBookingJSON` добавить поля `rating: row.rating ?? null`, `rating_comment: row.rating_comment ?? null`.
- В iOS в модели `Booking` добавить `rating: Int?`, `ratingComment: String?` и маппинг из snake_case.

---

### 6. Улучшение пустых состояний (низкий приоритет)

- В «Прошедшие» при пустом списке — не только текст «Нет прошедших записей», но и кнопка **«Перейти к услугам»** (переход на вкладку Услуги).
- В «Предстоящие» при пустом списке — кнопка **«Записаться»** (то же действие).

---

### 7. История обменов баллов (низкий приоритет, при наличии API)

- Если появится endpoint вида «последние обмены» (например, в профиле или отдельный), добавить в раздел лояльности блок **«История обменов»**: что и когда обменяно, сколько баллов списано.

---

## Итог по приоритетам

| № | Улучшение                    | Сложность | Польза для пользователя |
|---|------------------------------|-----------|---------------------------|
| 1 | Рейтинг после записи        | Средняя   | Высокая                   |
| 2 | Локальные напоминания        | Средняя   | Высокая                   |
| 3 | Записаться снова             | Низкая    | Средняя                   |
| 4 | Добавить в календарь         | Средняя   | Средняя                   |
| 5 | API: rating в списке записей | Низкая    | Нужно для п.1             |
| 6 | Пустые состояния с CTA      | Низкая    | Средняя                   |
| 7 | История обменов баллов      | Зависит от API | Средняя             |

Рекомендуемый порядок внедрения: **5 → 1** (рейтинг), затем **2** (напоминания), затем **3, 4, 6** по желанию.

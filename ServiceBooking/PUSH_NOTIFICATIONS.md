# Push-уведомления

## Что сделано в приложении

- **Разрешение:** запрашивается при включении «Push-уведомления» в профиле и при первом запуске (если уже включено).
- **APNs:** приложение регистрируется в Apple Push Notification service, device token отправляется на бэкенд (`PUT /api/v1/profile/push_token`).
- **Локальные напоминания:** планируются только при включённых push (напоминание о записи за 15 мин / 1 час / 1 день).
- **Нажатие на уведомление:** открывается вкладка «Мои записи» и показывается соответствующая запись (по `bookingId` в `userInfo`).
- **Показ в foreground:** уведомления отображаются баннером и в списке даже когда приложение открыто.

## Бэкенд (веб-консоль)

- В таблицу `clients` добавлено поле `apns_device_token` (миграция при старте).
- Эндпоинт `PUT /api/v1/profile/push_token` с телом `{ "push_token": "<hex-токен>" }` или `{ "push_token": null }` для снятия.

## Отправка push с сервера (готово в веб-консоли)

В веб-консоли реализована отправка push через APNs:

- **Сервис:** `web-console/services/push.js` (JWT + HTTP/2).
- **Когда отправляется:** при смене статуса записи на «Подтверждена», «В процессе», «Завершена» и при отправке сообщения клиенту из админки (раздел уведомлений).
- **Настройка:** переменные окружения `APNS_KEY_ID`, `APNS_TEAM_ID`, `APNS_BUNDLE_ID`, `APNS_KEY_PATH` (или `APNS_KEY`). Пример в `web-console/.env.example`.

Без этих переменных консоль работает как раньше; push не отправляются. В payload передаётся `bookingId` для перехода в приложении при нажатии на уведомление.

## Xcode

- В проекте включён **Push Notifications** через `ServiceBooking.entitlements` (`aps-environment`).
- Для **Archive / App Store** в Xcode: **Signing & Capabilities** → при необходимости добавьте возможность **Push Notifications** — Xcode подставит `production` для релиза.

# Другие рекомендации по проекту Service Booking

Идеи с другим фокусом: удобство, вовлечение, операции, качество продукта.

---

## Удобство и повторные визиты

| Идея | Описание |
|------|----------|
| **Избранные услуги** | В карточке услуги кнопка «В избранное». На главной — блок «Избранное» для быстрой повторной записи без поиска. |
| **«Часто заказывают вместе»** | Под деталями услуги показывать 1–2 другие популярные услуги (по данным с бэкенда или статично). |
| **Очередь / «Уведомить о свободном слоте»** | Если нужная дата занята — кнопка «Уведомить, когда появится место». Бэкенд при освобождении слота шлёт уведомление (или пуш). |
| **Повторяющаяся запись** | При создании записи опция «Повторять каждые 2 недели» (или раз в месяц) — создание серии записей или напоминание «пора записаться снова». |

## Вовлечение и лояльность

| Идея | Описание |
|------|----------|
| **Реферальная программа** | «Пригласи друга» — уникальная ссылка или код. За регистрацию друга — бонусные баллы обоим. В админке — учёт приглашённых. |
| **Прогресс до следующего уровня** | В разделе лояльности наглядная шкала: «До статуса „Постоянный клиент“ осталось 50 баллов» с визуальным прогресс-баром (частично уже есть — можно усилить). |
| **Промокод / скидка при записи** | Поле «Промокод» в форме записи. Бэкенд проверяет код и применяет скидку к цене или начисляет бонусные баллы. |
| **«Что нового» в приложении** | Экран или баннер после обновления: краткий changelog (1–3 пункта). Показывать один раз после установки новой версии. |

## Операции и админка

| Идея | Описание |
|------|----------|
| **Календарь записей** | В админке вид «Календарь» (день/неделя): записи по дням, удобно планировать загрузку. |
| **Заметки к клиенту** | В карточке клиента поле «Заметка» (только для админа): «Предпочитает утренние слоты», «Аллергия на …» — видно при работе с записью. |
| **Теги клиентов** | Метки: VIP, Новый, Неактивный. Фильтр по тегам в списке клиентов и при рассылке уведомлений. |
| **Правило отмены** | В настройках или на экране записи текст: «Отменить запись можно не позднее чем за 24 часа». При попытке отмены позже — предупреждение или запрет. |
| **Недоступные даты для услуги** | В веб-консоли возможность пометить дни/периоды, когда услуга не оказывается (отпуск, праздник). В приложении эти дни без слотов. |

## Качество и надёжность

| Идея | Описание |
|------|----------|
| **Проверка версии приложения** | Бэкенд отдаёт минимальную поддерживаемую версию (например, в `/settings` или `/company`). Если версия приложения ниже — показать экран «Обновите приложение» с ссылкой на App Store. |
| **Повтор при ошибке сети** | При неудачном запросе (таймаут, 5xx) автоматический повтор 1–2 раза с задержкой или кнопка «Повторить» крупнее и заметнее. |
| **Сохранение черновика записи** | Если пользователь выбрал услугу, дату и слот, но закрыл экран — при следующем открытии предложить «Продолжить запись?» с сохранёнными данными (локально). |
| **Логирование ошибок (без PII)** | Отправка анонимных отчётов о сбоях (тип ошибки, экран, версия ОС) на свой сервер или в сервис вроде Sentry — чтобы видеть проблемы пользователей. |

## Улучшение экранов

| Идея | Описание |
|------|----------|
| **Онбординг после первого входа** | Короткий тур: «Вот услуги», «Здесь ваши записи», «Профиль и баллы» — 2–3 экрана с возможностью пропустить. |
| **Длительность в выборе слота** | Рядом со временем слота показывать «30 мин» / «3 ч» по длительности услуги, чтобы было понятно, до какого времени занято. |
| **Поиск по записям** | В «Мои записи» поле поиска по названию услуги или дате — удобно при большом количестве записей. |
| **Фильтр новостей** | В новостях — по дате или «только непрочитанные», если бэкенд помечает прочитанное. |

## Нестандартные идеи

| Идея | Описание |
|------|----------|
| **Подарочный сертификат** | В админке создание сертификата (номинал или услуга). Клиент вводит код в приложении — сумма/услуга зачисляется на счёт или сразу доступна для записи. |
| **Виджет «Следующая запись»** | Виджет на главный экран iOS: дата и время следующей записи, название услуги. Без перехода в приложение видно, когда ехать. |
| **Face ID / Touch ID** | Опциональная блокировка входа в приложение по биометрии (например, только в профиле или при открытии приложения). |
| **Тёмная тема по расписанию** | Кроме ручного переключения темы — «Авто: по системе» и опция «Тёмная с 22:00 до 7:00» для комфорта ночью. |

---

Можно выбирать 1–2 пункта из любого блока и внедрять по очереди — так продукт будет развиваться в разных направлениях без дублирования прошлых рекомендаций.

# Рекомендации по дополнениям проекта Service Booking

Краткий список идей по приоритету: что уже сделано и что можно добавить дальше.

---

## Уже реализовано (iOS)

- Рейтинг после записи (UI + локальное хранение оценённых)
- Локальные напоминания о записи (UNUserNotificationCenter)
- Кнопка «Записаться снова» + переход на услугу
- Добавить запись в календарь (EventKit)
- Пустые состояния с кнопками «Записаться» / «Перейти к услугам»
- Заглушка «История обменов» в разделе лояльности

---

## Рекомендуемые дополнения

### Высокий приоритет

| Дополнение | Где | Зачем |
|------------|-----|--------|
| **Рейтинг в API** | Веб-консоль | В `toBookingJSON` (bookings.js) добавить `rating` и `rating_comment`. Тогда в iOS можно показывать «Вы оценили: ★★★★» с сервера и не дублировать логику в UserDefaults. |
| **Push-уведомления (remote)** | iOS + бэкенд | Отправка пушей при смене статуса записи, новых сообщениях от админа, новостях. Нужен APNs + сохранение device token на бэкенде и endpoint для отправки. |
| **HTTPS и базовая защита** | Веб-консоль | В продакшене — HTTPS, заголовки безопасности (CORS, X-Content-Type-Options), по желанию rate limiting на API. |

### Средний приоритет

| Дополнение | Где | Зачем |
|------------|-----|--------|
| **История обменов баллов** | Веб-консоль + iOS | API вида `GET /profile/redemptions` или поле в профиле с последними обменами. В iOS — блок в разделе лояльности вместо заглушки. |
| **Экспорт записей (CSV/Excel)** | Веб-консоль | В админке кнопка «Скачать отчёт» по записям за период — для бухгалтерии и аналитики. |
| **Простой дашборд** | Веб-консоль | Сводка: записей за день/неделю, выручка, популярные услуги. Удобно для владельца. |
| **Офлайн-режим (только чтение)** | iOS | Кэш списка услуг и профиля в локальной БД (например, SwiftData/UserDefaults). При отсутствии сети — показ закэшированных данных и явная пометка «Офлайн». |
| **Глубокие ссылки** | iOS | Поддержка URL-scheme или Universal Links: открытие конкретной записи или экрана «Услуги» из письма/смс. |
| **Локализация** | iOS | В Info есть ru/en — вынести все строки в Localizable.xcstrings и проверить отображение на обоих языках. |

### Низкий приоритет

| Дополнение | Где | Зачем |
|------------|-----|--------|
| **Haptic feedback** | iOS | Лёгкая вибрация на ключевых действиях (подтверждение записи, успешная оценка). |
| **Поделиться записью** | iOS | Share sheet: отправить текст/ссылку о предстоящей записи в мессенджер или почту. |
| **Резервное копирование БД** | Веб-консоль | Периодический бэкап SQLite (cron + копия в облако или скачивание из админки). |
| **API-документация** | Веб-консоль | OpenAPI (Swagger) или статичный Markdown со списком endpoints для разработчиков. |
| **Логирование действий админа** | Веб-консоль | Таблица или лог: кто и когда изменил статус записи, отправил уведомление — для аудита. |

## Быстрые победы (мало труда, заметная польза)

1. **Веб-консоль:** добавить в `toBookingJSON` поля `rating` и `rating_comment` — сразу улучшится согласованность рейтинга с сервером.
2. **iOS:** запрашивать разрешение на уведомления при первом выборе напоминания в форме записи (если ещё не запрашивали).
3. **Веб-консоль:** в админке на странице записей — фильтр по дате «сегодня» по умолчанию.
4. **Оба:** добавить в README или DEPLOY раздел «Переменные окружения» (PORT, DB_PATH, и т.д.) для деплоя.

---

## Не обязательно, но полезно

- **Siri Shortcuts:** действие «Записаться на услугу» — открывает приложение на экране услуг.
- **Виджет «Следующая запись»:** на главный экран (у вас уже есть виджет для Live Activity — можно отдельно карточку «ближайшая запись»).
- **A/B или фичевые флаги:** если позже появятся эксперименты с интерфейсом или логикой.

---

Итог: в первую очередь имеет смысл добавить **рейтинг в API** и по возможности заложить фундамент под **push-уведомления**; остальное — по мере приоритетов продукта и времени на разработку.
